# Rime Engine Configuration

The Rime engine reads its settings from the Typio host configuration under the
`[engines.rime]` table. All keys are optional.

## Config keys

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `schema` | string | `""` | Schema id to use (e.g. `luna_pinyin`). If empty, the first available schema is selected automatically. The engine persists the user’s choice back to this key. |
| `shared_data_dir` | string | `/usr/share/rime-data` | Directory containing system Rime data (`default.yaml`, `.schema.yaml` files, etc.). Compiled-in default can be overridden at build time with `-DTYPIO_RIME_SHARED_DATA_DIR`. Supports `~` expansion. |
| `user_data_dir` | string | `<typio-data-dir>/rime` | Directory for per-user Rime configuration and build artifacts (`default.custom.yaml`, `build/`, etc.). Supports `~` expansion. |

## Paths in practice

- **Shared data** — Usually provided by the distribution package `rime-data`.
  Debian/Ubuntu: `/usr/share/rime-data`

- **User data** — Defaults to a `rime/` subdirectory inside Typio’s data
directory. If Typio places its data under `~/.local/share/typio`, the user
Rime directory becomes `~/.local/share/typio/rime/`.

  Inside that directory you can create or edit:
  - `default.custom.yaml` — overrides for `default.yaml`
  - `<schema>.custom.yaml` — per-schema overrides
  - `build/` — generated by Rime deployment (do not edit manually)

## Example `typio.toml`

```toml
[engines.rime]
schema = "luna_pinyin"
user_data_dir = "~/.local/share/typio/rime"
```
