# Rime Engine Configuration

The Rime engine reads its settings from the Typio host configuration under the
`[engines.rime]` table. All keys are optional.

## Config keys

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `schema` | string | `""` | Schema id to use (e.g. `luna_pinyin`). If empty, the first available schema is selected automatically. The engine persists the user's choice back to this key. |

## Data directories

Data directories are no longer user-configurable. They are resolved
automatically:

- **User data** — provided by the host via `typio_instance_get_engine_data_dir()`.
  Defaults to `~/.local/share/typio/rime/`. The engine manages its own internal
  layout within this directory (`default.custom.yaml`, `build/`, etc.).

- **Shared data** — compile-time constant `TYPIO_RIME_SHARED_DATA_DIR`, defaulting
  to `/usr/share/rime-data`. Overridable at build time via `-D` flag. This
  contains system schemas and dictionaries installed by the distribution package.

## Paths in practice

- **Shared data** — Usually provided by the distribution package `rime-data`.
  Debian/Ubuntu: `/usr/share/rime-data`

- **User data** — `~/.local/share/typio/rime/` by default.
  Inside that directory you can create or edit:
  - `default.custom.yaml` — overrides for `default.yaml`
  - `<schema>.custom.yaml` — per-schema overrides
  - `build/` — generated by Rime deployment (do not edit manually)

## Example `core.toml`

```toml
[engines.rime]
schema = "luna_pinyin"
```
